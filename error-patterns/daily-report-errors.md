## 已记录的教训

### 2026-04-14

#### 重复会话文件导致上下文碎片化
**错误做法**：存在2个会话文件 daily-report_acb6f4df.json 和 daily-report_eef05a65.json
**原因**：配置变更或重启时创建了新会话
**正确做法**：
- 定期检查 sessions 目录
- 删除重复的会话文件
- 配置 share_session_in_channel = true
**教训**：每次配置变更后检查 sessions 目录

#### 过度输出中间过程
**错误做法**：给用户展示每个执行步骤
**原因**：默认详细输出模式
**正确做法**：只输出最终结果，必要确认才提示
**教训**：询问用户偏好，按需输出

#### 消息漏读/不响应
**错误做法**：用户发送消息后机器人无响应
**原因**：
- 缺少 reply_to_trigger 配置
- 缺少 receive_channel_messages_only 配置
- 会话切换导致历史丢失
**正确做法**：
- 配置 share_session_in_channel = true
- 配置 reply_to_trigger = true
- 配置 receive_channel_messages_only = true
**教训**：飞书机器人必须配置这三个参数
